    How-to guides¶

    Recipes for solving specific operational tasks. These assume you already have kagents installed and at least a basic working AgentTeam. If not, start with the Getting Started tutorial.

    Install¶

    Cloud-specific install paths covering the ReadWriteMany storage configuration that's the actual deployment friction point on each cloud:

    • Install on Amazon EKS. EFS CSI driver + EFS file system + Access Points
    • Install on Google GKE. Filestore CSI driver + Filestore instance
    • Install on Azure AKS. Azure Files CSI driver + Premium NFS share

    Each guide ends with the same make mailbox-smoke-test verification step.

    Operate¶

    Day-to-day operational tasks once kagents is running:

    • Expose the dashboard. Port-forward for dev, Ingress with basic auth for prod, oauth2-proxy for corporate SSO, namespace-scoping
    • Configure shared storage. Sizing the team-state / repo / output PVCs, backup strategies per cloud backend, performance tuning recipes
    • Set budget alerts. Per-team budgetLimit, chart-wide default, webhook events to Slack/PagerDuty, Prometheus alert rules

    Looking for something else?¶

    • New to the project? Start with the Getting Started tutorial.
    • Want to understand how it works? See the Explanation section.
    • Need a specific CRD field or Helm value? See the Reference section.
